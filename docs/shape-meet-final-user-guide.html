Guia de usuario final

Shape Meet

Flujo esperado para operar una reunion con identidad local: el admin asigna modelos, la app desktop los descarga y valida, el host activa rostro, voz y fondo, y los invitados entran por enlace web.

Resumen Roles Modelos Admin Host Invitado Soporte

Resumen del producto

Shape Meet es una app de videollamadas propia. El host autorizado usa una app Tauri en su PC, procesa rostro, voz y fondo localmente, crea una reunion y comparte un enlace. Los invitados entran desde navegador movil o desktop sin instalar la app.

Admin

Prepara identidades

Crea usuarios, promueve hosts, sube o registra paquetes de identidad y asigna versiones a cada host.

Desktop

Procesa localmente

Descarga solo modelos autorizados, valida checksum y publica tracks procesados por LiveKit.

Invitado

Entra por link

Abre la sala publica, espera admision si aplica y recibe audio y video ya procesados.

Roles y permisos

Rol Puede hacer No deberia hacer
Administrador Crear usuarios, asignar rango HOST, publicar identidades, revocar paquetes y revisar auditoria. Compartir secretos, tokens de storage o credenciales.
Host Iniciar sesion desktop, descargar identidades asignadas, probar camara/microfono, crear reuniones y admitir invitados. Subir modelos arbitrarios desde la app final.
Invitado Entrar por navegador, activar camara/microfono y participar en la sala. Instalar runtime IA o descargar modelos.

Como se manejan los modelos

La app no debe buscar modelos libremente. El admin asigna una version concreta a un host y Tauri descarga exactamente ese paquete, valida SHA-256/tamano y lo activa localmente.

Confirmacion clave. Los modelos entrenados para vcclient suelen venir como .pth y .index. Esos archivos son de voz. El cambiador de rostro normalmente no espera esos archivos; espera una foto fuente o un paquete facial especifico del motor elegido. BackgroundMattingV2 usa un checkpoint .pth base y una captura local de fondo limpio.
Modulo Archivos esperados Quien los recibe
Voz model.pth, model.index, config vcclient000 / w-okada / RVC
Rostro rapido Foto fuente .jpg o .png FaceFusion o motor compatible
Rostro entrenado Formato propio del motor facial final DFM/otro motor, si se aprueba licencia
Fondo Checkpoint BMV2, clean plate, assets de fondo Runtime local de fondo

Repositorio de paquetes

Para piloto se puede usar un repositorio de GitHub Releases con assets versionados. El admin debe guardar URL, version, tamano, SHA-256 y host asignado. Si el repositorio es privado, el backend debe mediar la descarga; la app no debe llevar tokens GitHub embebidos.

La actualizacion de la app Tauri y la descarga de modelos son canales distintos. Tauri updater sirve para actualizar la app firmada; los modelos deben tener manifest, cache, validacion y revocacion propios.

Flujo del administrador

Crear o revisar usuario

El administrador crea el usuario con correo, rango y estado. Solo usuarios con rango HOST pueden operar la app desktop como host.

Preparar identidad

Se crea una identidad con nombre, version y tipo. Puede ser foto facial, identidad entrenada o paquete abierto, segun el motor.

Subir o registrar paquete

El admin sube el artefacto al storage o registra la URL del release asset. El sistema guarda hash, tamano y version.

Publicar para el host

La identidad pasa a disponible y queda asignada al host. Desde ese momento la desktop puede verla y descargarla.

Revocar si hace falta

Al revocar, la identidad desaparece del host y la desktop debe retirar la cache local en el siguiente refresh.

Flujo del host en desktop

Iniciar sesion

El host abre la app Shape Meet, entra con su usuario autorizado y confirma que la app apunta al entorno correcto.

Elegir identidad

La app lista identidades asignadas. Si una identidad no esta en cache, muestra descarga requerida antes de activarla.

Descargar y validar

Tauri descarga el paquete autorizado, valida SHA-256 y tamano, y guarda el package en la cache local del equipo.

Preparar equipo

El host selecciona camara y microfono. La app revisa GPU, sidecar, runtime IA y motores disponibles.

Activar rostro, voz y fondo

El host activa los toggles que correspondan. Si usa fondo, captura el fondo limpio antes de entrar o mientras prepara la reunion.

Probar identidad

La prueba procesa un frame y un chunk de audio. Si algo falla, rostro/voz/fondo muestran el estado accionable sin bloquear toda la app.

Crear reunion

El host crea o abre una reunion, copia el enlace y lo envia al invitado. Al entrar, la app publica los tracks procesados.

Operar la llamada

El host admite invitados, controla microfono/camara y puede abrir diagnostico si necesita soporte. La superficie principal debe seguir viendose como una videollamada normal.

Estados visibles recomendados

Estado Texto para usuario Resultado
download_required Descargar identidad El host descarga antes de activar rostro o voz.
ready Lista para usar Se habilitan toggles y preflight.
runtime_missing Runtime IA incompleto Se abre diagnostico con motor faltante.
checksum_failed Descarga no valida No se activa el modelo; se permite reintentar.
revoked Identidad retirada Se oculta la identidad y se elimina cache local.
Estado final correcto. La app puede publicar un track llamado procesado aunque el pipeline este usando fallback. Para decir que la identidad esta activa, el diagnostico debe confirmar LiveKit conectado, track publicado, ultimo frame/audio en estado processed, procesador real sin passthrough y media fresca dentro de la ventana de latencia esperada.

Flujo del invitado

Abrir enlace

El invitado abre el link de reunion en navegador movil o desktop. No instala la app Tauri ni descarga modelos.

Entrar con nombre

El invitado escribe su nombre, concede permisos de camara y microfono si los usara, y espera admision cuando aplique.

Participar

LiveKit entrega el video y audio publicados por el host. Si el host activo IA, el invitado recibe el track procesado, no el preview local sin procesar.

Soporte y diagnostico

Equipo

PC AMD

Sirve para instalar app, login, crear reunion, probar UI, LiveKit y fallback. No valida modelos pesados de rostro/voz/fondo.

Equipo

PC RTX

Valida CUDA, VRAM, FaceFusion/BMV2/vcclient000, latencia real y publicacion de tracks procesados a invitados.

Debug

Bundle de soporte

Debe incluir logs, runtime, version de package, GPU y errores. No debe incluir frames, audio, fotos fuente ni secretos.

Licencia. Si el motor facial usa InSwapper, InsightFace u otro componente non-commercial, no se debe entregar a clientes ni usar comercialmente sin licencia aprobada. El package debe marcar si el uso comercial esta permitido.
Privacidad. Los modelos de identidad son datos sensibles. El admin asigna, la app descarga solo lo autorizado y la cache local debe poder revocarse.
